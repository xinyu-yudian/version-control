     - 取消：终止操作，不执行任何提交。
  3. 复合指令：同时包含「检查：」与「提交：」时，先输出完整检查结果，再唤起提交弹窗。

## 工作流程**严格按照以下步骤执行**
### 步骤1：公共前置预处理（复合指令仅执行1次，所有分支共享）
1. 解析用户完整指令，拆分两类目标：
   - 查询目标：`检查：` 后的目录列表，多目录以「、」分隔
   - 提交目标：`提交：` 后的内容，以「 | 」分隔为默认备注、限定提交目录列表；无分隔符则全部视为默认备注，提交范围为对应仓库全部变更
2. 逐个目录执行**短名映射替换**：
   -规则说明：
      1. 目标目录若在工作区范围内，向上查找最多到工作区根目录即终止；
      2. 目标目录若不在工作区范围内，仅检测目标目录自身，不执行向上递归。
   - 校验目录物理存在性，不存在则标记异常并提示跳过
3. 逐个检测**目标目录对应版本控制系统（Git / SVN）**，全局缓存检测结果：
   - 检测优先级：优先检测目标目录自身，自身无仓库再逐级向上遍历父目录（不超出工作区）
   - 第一步：检查目标目录自身是否存在 `.git` 文件夹（Git）或 `.svn` 文件夹（SVN），存在则直接标记目标目录为「仓库根目录」，终止检测
   - 第二步：自身无仓库时，从目标目录的父级开始逐级向上遍历，**每一级目录必须同时检测 `.git` 和 `.svn` 两类仓库**：
     - 找到第一个包含任意一类版本库的目录，立即标记为「仓库根目录」，终止遍历
     - 遍历到工作区根目录仍未找到版本库，终止遍历，标记为「无仓库」
   - 缓存维度：目标目录路径、仓库类型、仓库根路径、目标目录相对仓库根的路径
4. 按目录分组扫描对应仓库变更，全局缓存结果；所有 Git/SVN 命令均以**仓库根路径**为执行基准，同时**作用范围严格限定为用户指定的目标目录**：
   - 通用缓存：目标目录范围内的变更文件列表、各文件全量diff内容、变更类型；
   - Git仓库额外缓存：目标目录范围内对应的未推送提交列表（提交哈希、提交信息、提交时间）。
5. 区分当前指令类型：仅查询 / 仅提交 / 查询+提交复合指令，按规则顺序执行对应分支。

### 步骤2.1：包含「检查：」时执行（查询输出）
1. 从全局缓存按目录读取变更文件，**仅返回目标目录范围内的变更，不输出仓库其他目录内容**
2. 复用缓存内各文件的完整diff内容；
3. 严格遵循**输出格式规则**中的输出模板输出；无版本仓库的目录单独输出提示信息。息。

### 步骤2.2：包含「提交：」时执行（弹窗提交流程）
1. 从缓存读取默认提交备注、限定目标目录、预扫描结果，先判定提交场景：
   - 场景A：存在未提交的文件变更 → **仅加载目标目录范围内的变更文件**填充预览区，备注输入框可编辑，走「commit + push」流程
   - 场景B：无未提交文件，但存在未推送的本地提交 → 加载待推送提交记录填充预览区，备注输入框置灰不可编辑，走「仅 push」流程
   - 场景C：既无未提交文件、也无未推送提交 → 直接输出“无待提交变更”，终止提交流程
2. 唤起交互弹窗等待用户操作，严格遵循**弹窗交互规范**；
3. 捕获命令执行日志，严格遵循**输出格式规则**中的输出模板输出；
4. 异常兜底：冲突、权限不足、服务器离线等错误完整输出原始日志。

### 复合指令场景（同时存在「检查：xxx」+「提交：xxx」）执行顺序
**必须严格按「先执行步骤2.1，再执行步骤2.2」的顺序执行**

## 输出格式规则
- 匹配输出模版：
  1. 包含「检查：」的指令（含复合指令场景）：严格遵循 `references/formats-check.md` 中的模板输出
  2. 包含「提交：」的指令（含复合指令场景）：严格遵循 `references/formats-push.md` 中的模板输出
- 默认路径输出使用**相对仓库路径**，过滤无任何变更的文件；重命名、权限变更等特殊变更按规则正常展示。
- 多目录场景按目录分组输出，每组标注对应目录路径与版本控制系统；无版本仓库的目录单独输出跳过提示。
- **Diff 展示强制规则**：
  1. 以文件为单位聚合展示，单个文件默认仅展示一个折叠diff块，不按零散变更块拆分多个条目；
  2. 仅当单个连续变更块（hunk）的**纯变更行数（新增行数+删除行数，不含上下文行）**超过50行时，才将该超大变更块单独拆分为子块；
  3. 每个变更块首尾必须包含上下各5行未修改的上下文代码，文件首尾不足5行时按实际行数展示。
  4. 行内高亮 diff 展示统一调用「统一 Diff 格式命令」的输出结果解析渲染，左右对比格式调用「批量导出前后内容命令」的输出结果解析渲染。
- 自动区分逻辑分支加载对应命令：
  - 检查逻辑：按目录所属版本控制系统，分别加载 `references/commands-git-check.md` / `references/commands-svn-check.md`
  - 提交逻辑：按目录所属版本控制系统，分别加载 `references/commands-git-push.md` / `references/commands-svn-push.md`
- 执行命令完成后统一解析、渲染标准化输出。

## 弹窗交互规范（仅「提交：」指令触发）
弹窗分为三块区域：
1. **备注输入框**
   单行可编辑文本，默认值 = 用户指令「提交：」后方的备注文本，支持手动修改、清空；
   点击确认时校验：去除首尾空白字符后不可为空，否则弹窗提示填写有效备注。
2. **变更预览只读输出框**
   弹窗加载时自动预执行未提交查询，展示本次待提交的全部文件；
   若无任何待提交文件，预览框提示「当前无待提交变更，无法提交」，确认按钮置灰禁用。
3. **操作按钮**
   - 【确认提交】：校验通过后执行提交推送命令；
   - 【取消】：关闭弹窗，终止全部流程，输出取消提示。
